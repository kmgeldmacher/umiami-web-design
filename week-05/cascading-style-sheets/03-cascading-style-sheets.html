Cascading Style Sheets

CSS or stylesheets is/are the method used to apply presentational elements to an HTML document.

This includes: fonts, colors, positioning, decorations, etc.

As you can see with this page, our body text is styled in a serif font, while headlines are styled in a sans-serif font.

Why 'cascading'?

Because certain styles applied to a 'parent' element are inherited by 'child' elements.

How to add styles

We can style our document by:

  1. adding the style attribute to an element (inline style)
  2. adding our styles within <style></style> tags between the <head></head> tags of our HTML document
  3. adding a link to an external style sheet

Which way is best?

No. 3!

There may be rare cases where you need to use method No. 1 or No. 2. But if you are designing a web site from scratch, always include external stylesheets.

We do this by adding the following line of code between our <head></head> tags:

<link rel="stylesheet" href="css/screen.css" />

<link> defines a relationship between the document and an external resource, in this case, a stylesheet.

rel="stylesheet" tells the browser our external resource is a stylesheet.

href="css/style.css" is the location of our stylesheet (css/style.css is a value that can be changed).

Why not No. 1 and No. 2?

Let's say you maintain a site with 100,000 pages. You apply an inline style to every paragraph tag on a page that uses the font Helvetica. And you include style tags in the head of each page that turns every link green.

Your boss comes to you and says, "I hate Helvetica! And I hate green!"

Now imagine you have one external style sheet that looks like this:

p { 
  font-family: "Helvetica", sans-serif; 
}

a {
  color: green;
}

Isn't it going to be nicer to change two lines of code instead of hunting for each style setting across 100,000 pages?
